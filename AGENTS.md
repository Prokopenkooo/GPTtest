# AGENTS.md

## Назначение репозитория

Этот репозиторий используется как тестовая площадка для работы с Codex и подготовки будущих программных решений ПРОЛЕКС.

Основной фокус:
- автоматизация инженерных и бизнес-процессов;
- разработка утилит и прототипов;
- подготовка к будущим проектам по КОМПАС-3D, ЛОЦМАН PLM, ВЕРТИКАЛЬ и ПОЛИНОМ MDM;
- аккуратная, понятная, масштабируемая разработка.

## Правила работы Codex

- Перед изменениями сначала изучи структуру проекта.
- Не переписывай весь проект без необходимости.
- Делай минимальные точечные изменения.
- Если задача неоднозначна, сначала задай уточняющие вопросы.
- После изменений объясни, что именно изменено и зачем.
- Не добавляй зависимости без обоснования.
- Не добавляй секреты, токены, пароли, API-ключи или коммерческие данные.
- Не используй реальные данные заказчиков.
- Не создавай хаотичные файлы без структуры.

## Структура проекта

Пока структура минимальная:

- `README.md` — описание проекта.
- `AGENTS.md` — инструкции для Codex.
- `src/` — исходный код, если появится.
- `docs/` — документация и заметки.
- `tests/` — тесты.

## Команды проверки

Если проект на C# / .NET:

```bash
dotnet build
dotnet test
Если проект на Python:
python -m pytest

Если тестов пока нет, Codex должен явно написать, что автоматическая проверка отсутствует.

Критерий готовности

Задача считается выполненной, если:

код или документы изменены минимально и по делу;
изменения понятны;
проект не содержит секретов;
при наличии тестов они запущены;
Codex объяснил результат человеческим языком.
